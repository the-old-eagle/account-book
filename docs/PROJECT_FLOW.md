# account-book 项目全流程

> 适用于 account-book 私有仓库（记账网页：Excel 数据源 + GitHub Pages 部署）。
> 本文档定义从需求到交付的标准化执行流程，全程在云端沙箱自动化执行。

---

## 1. 全流程六步

整个项目按以下六个阶段顺序推进，前序阶段产物作为后序阶段输入：

```
需求 → 开发 → 测试 → 修 Bug → 合并 → 交付
```

| 阶段 | 目标 | 关键产物 |
|---|---|---|
| 1. 需求 | 输出完整 PRD 需求文档 | PRD（功能清单、数据结构、交互、验收标准） |
| 2. 开发 | 按 PRD 切分 feature 分支并实现 | 功能分支代码 + 规范 commit |
| 3. 测试 | 生成并运行自动化测试用例 | 测试用例 + 测试报告 |
| 4. 修 Bug | 自动定位并修复测试发现的问题 | 修复 commit + 复测报告 |
| 5. 合并 | 测试通过后将功能分支合并回 main | main 分支最新提交 |
| 6. 交付 | 生成最终仓库链接与发布产物 | 仓库 URL + GitHub Pages 部署 |

## 2. 执行顺序（严格串行）

1. **先输出完整 PRD 需求文档**，明确模块划分、数据来源（Excel）、页面交互与验收标准；
2. **再开发**：依据 PRD 切分 `feature/模块名` 分支，逐模块实现并提交规范 commit；
3. **再生成并运行自动化测试用例**：覆盖核心功能与边界场景；
4. **自动修复 Bug**：测试失败时自动定位、修复并复测，直至全绿；
5. **测试通过后合并**：将该轮所有功能分支合并到 `main`；
6. **最终交付**：产出可访问的仓库链接与 GitHub Pages 部署。

## 3. 飞书进度推送

每完成一个大阶段，通过飞书 Webhook 推送进度与对应产物链接。四大推送节点：

| 节点 | 触发时机 | 推送内容 |
|---|---|---|
| 需求完成 | PRD 文档定稿 | PRD 摘要 + 文档链接 |
| 开发完成 | 所有 feature 分支实现完毕 | 功能模块清单 + 分支链接 |
| 测试完成 | 自动化测试全部通过 | 测试用例数 + 通过率 + 报告链接 |
| 最终交付 | main 合并与部署完成 | 最终仓库链接 + Pages 地址 |

- 推送格式：post 富文本（标题 + 多行正文）。
- 推送工具：`/workspace/.sandbox/feishu/send_feishu.py`。

## 4. 异常处理与重试

- 任何阶段（开发、测试、合并、推送）出现异常，**自动重试 3 次**，重试间隔指数退避（1s / 2s / 4s）。
- 3 次重试仍失败，则**立刻推送告警到飞书群**，告警内容含失败阶段、错误原因与时间。
- 告警推送本身也重试 3 次；告警仍失败则以非零退出码退出并记录日志。

## 5. 执行环境约束

- **全程在云端沙箱执行，禁止访问本地电脑文件。**
- 所有代码、文档、产物仅落在沙箱工作目录（`/workspace` 及其子目录）。
- 不读取、不上传用户本机任意路径下的文件。
- 不修改 `/workspace/.trae/specs/` 下任何文件。
- 文档与推送内容中**不得出现任何 GitHub Token 或密钥**。

## 6. 分支合并策略

- 遵循《分支规则》（BRANCH_RULES.md）：`main` 为受保护分支，仅接受合并。
- **每轮 feature 开发与测试通过后，自动将该轮所有功能分支合并到 `main` 主分支**，采用 `--no-ff` 保留分支历史。
- 合并完成后生成**最终仓库链接**，作为交付节点产物之一推送至飞书。
- 合并后可清理已合并的功能分支。

## 7. 交付物清单

最终交付应包含：

1. 完整 PRD 需求文档；
2. 可运行的记账网页源码（main 分支）；
3. 自动化测试用例与测试报告；
4. GitHub Pages 部署地址；
5. 最终仓库链接；
6. 飞书群内四大阶段进度推送记录。

## 流程速览

```
[需求] 输出 PRD ──(飞书: 需求完成)──┐
                                    ▼
[开发] 切 feature 分支实现 ──(飞书: 开发完成)──┐
                                               ▼
[测试] 生成并运行测试用例
   │   ├─ 失败 → 自动修 Bug → 复测（重试 ≤3）
   │   └─ 通过 ──┐
   ▼             ▼
[修 Bug] [合并] 测试通过 → 合并所有 feature 到 main ──(飞书: 测试完成)
                                    │
                                    ▼
[交付] 生成仓库链接 + Pages 部署 ──(飞书: 最终交付)
```
