## 1. 产品定位

account-book 是一个面向**个人/小团队**的记账网页，部署于 **GitHub Pages**，为**纯静态站点**（无后端服务）。产品提供：

- 账目流水录入与展示。
- 通过 **前端模拟登录 + 角色权限** 区分「管理员」与「普通用户」两类使用角色。
- 外部 Excel 导入能力（SheetJS 解析）。
- 浏览器本地持久化（localStorage），按用户/设备隔离。

**重要约束**：前端模拟登录为非真正鉴权，仅用于角色区分；安全性有限，详见第 9 章「安全限制声明」。

---

## 2. 目标用户

| 角色 | 权限范围 |
| --- | --- |
| **管理员** | 可编辑 localStorage 内账目（增 / 删 / 改）+ Excel 导入 + 只读查看 |
| **普通用户** | 仅 Excel 导入 + 只读查看，**不可编辑**账目 |

- 角色凭证配置在前端配置文件 `auth.config.js`（账号/角色映射）。
- 登录为前端模拟，无服务端鉴权。

---

## 3. 核心功能（v1）

### 3.1 前端模拟登录
- 登录页提供账号/密码输入。
- 凭证来自前端配置文件 `auth.config.js`（账号 → 角色映射）。
- 登录态保存在浏览器内存 / sessionStorage，刷新后保持登录。
- **无真正服务端鉴权**，密码明文/哈希在前端可见，仅作角色区分。

### 3.2 角色权限控制
- 管理员：账目增 / 删 / 改 + Excel 导入 + 只读查看。
- 普通用户：Excel 导入 + 只读查看，禁用编辑入口。
- 前端按角色渲染可见操作（按钮/入口的显示与禁用）。

### 3.3 Excel 导入（只导入不回写）
- 用户可上传外部 `.xlsx` 文件。
- 前端用 **SheetJS** 解析，按字段映射（见第 4 章）写入 localStorage。
- **不实现**向仓库内 `data/account-book.xlsx` 回写。
- 导入策略：追加（保留既有数据）或覆盖，由导入时选项决定（v1 至少提供追加）。

### 3.4 账目流水展示
- 列表展示所有账目（来自 localStorage）。
- 筛选：按**类型**（收入/支出）、**分类**、**日期**（范围）筛选。
- 排序：默认按日期倒序。

### 3.5 分类汇总
- 按分类合计收入/支出金额。
- 展示各分类小计与总收入/总支出/结余。

### 3.6 收支图表（Chart.js）
- **趋势图**：按日期/月度展示收入与支出趋势。
- **分类占比图**：饼图/环形图展示各分类金额占比。

### 3.7 localStorage 本地持久化
- 所有账目数据保存在浏览器 localStorage。
- **按用户/设备隔离，不跨设备同步**。
- 刷新或重新打开站点后数据仍在；清浏览器数据将丢失。

### 3.8 种子数据载入
- 首次访问时，可一键载入仓库内 `data/account-book.xlsx` 作为示例/种子数据。
- 载入后写入 localStorage；用户可清空重新载入。

## 4. 数据源与存储

### 4.1 主存储
- **浏览器 localStorage**（按用户/设备隔离，不同步）。
- Key 命名建议：`account_book_records`（账目数组）。

### 4.2 种子/示例数据
- 仓库内 `data/account-book.xlsx`，仅作**种子/示例**，不作为运行时主存储。
- 首次访问可载入。

### 4.3 字段定义

| 字段 | 说明 | 取值 |
| --- | --- | --- |
| `date` | 日期 | `YYYY-MM-DD` 字符串 |
| `type` | 类型 | `收入` \| `支出` |
| `category` | 分类 | 字符串（如 餐饮 / 工资 / 交通） |
| `amount` | 金额 | number，单位元，正数 |
| `note` | 备注 | 字符串（可空） |

## 5. 技术选型

- **静态站点**：原生 HTML / JS / CSS，**无构建步骤**。
- **SheetJS**（xlsx）：Excel 解析，CDN 引入。
- **Chart.js**：收支图表，CDN 引入。
- 依赖均通过 CDN `<script>` 引入，无需 npm/打包。

---

## 6. 非功能需求

- 可经 **GitHub Pages** 访问（仓库 `main` 分支 `/root` 或 `/docs`）。
- **响应式布局**，适配桌面与移动端。
- **中文 UI**。
- **无后端依赖**，全部逻辑在前端。

---

## 7. v1 范围

包含以下核心功能：

1. 前端模拟登录（凭证来自 `auth.config.js`）。
2. 角色权限控制（管理员 / 普通用户）。
3. Excel 导入（SheetJS 解析 → 写入 localStorage）。
4. 账目流水展示（列表 + 类型/分类/日期筛选）。
5. 分类汇总（收入/支出按分类合计）。
6. 收支图表（Chart.js 趋势/分类占比）。
7. localStorage 本地持久化。
8. 种子数据载入（首次访问可载入 `data/account-book.xlsx`）。

---

## 8. v1 非范围（明确不做）

- **不实现**服务端鉴权（登录为前端模拟）。
- **不回写**仓库内 Excel（`data/account-book.xlsx` 只作种子）。
- **无**多设备同步（localStorage 按设备隔离）。
- **无**导出功能（v1 不提供导出 Excel/CSV）。
- **无**多用户账号管理后台。
- **无**服务端持久化。

---

## 9. 安全限制声明

> ⚠️ **前端模拟登录非真正鉴权，仅适用于低敏感场景。**

- 账号/角色凭证配置在前端 `auth.config.js`，**前端可见，可被查看/绕过**。
- 无服务端校验，**不能防止有技术能力者绕过登录或篡改角色**。
- localStorage 数据明文存储于浏览器，**清浏览器数据即丢失**。
- **UI 与 README 须显式提示用户**：不要在系统中存放敏感/重要财务数据；本系统不保证数据安全与隐私。
