# account-book

> 记账网页 —— 基于 Excel 数据源 + GitHub Pages 的轻量个人/家庭记账应用。

## 项目简介

`account-book` 是一个纯前端的记账网页应用，目标是用最低的部署成本（GitHub Pages 托管静态站点）实现日常收支记录、分类统计与可视化展示。所有业务逻辑运行在浏览器端，无需后端服务，开箱即用。

核心能力：

- 收支流水录入、编辑、删除
- 按分类 / 日期 / 账户维度汇总
- 图表可视化展示
- 从仓库内置的 Excel 种子文件加载初始数据
- 本地浏览器持久化保存用户数据

## 技术栈

| 类别 | 选型 | 说明 |
| --- | --- | --- |
| 站点类型 | 纯静态站点 | 直接由 GitHub Pages 托管，无构建/无服务端 |
| 数据解析 | SheetJS (xlsx) | 读取仓库内置 Excel 种子文件作为初始数据源 |
| 图表 | Chart.js | 渲染收支分类、趋势等可视化图表 |
| 存储 | localStorage | 浏览器本地持久化用户数据 |
| 部署 | GitHub Pages | 推送到 `main` 分支即可自动发布 |

## 数据存储说明

本应用采用「**种子数据 + 本地持久化**」的混合策略：

1. **Excel 种子数据（仓库内）**：仓库内置示例 Excel 文件，提供分类字典与示例流水，作为首次访问的初始化数据源，使用 SheetJS 在浏览器端解析。
2. **localStorage 本地持久化**：用户在网页中录入、修改的所有数据均保存在浏览器 `localStorage` 中，刷新或关闭页面后不会丢失。
3. **数据不离开浏览器**：除首次加载种子 Excel 外，用户数据不会被上传到任何服务器；清除浏览器缓存将导致数据丢失，请自行导出备份。

## 角色权限

应用内置两种角色，用于控制页面功能可见性：

| 角色 | 权限 |
| --- | --- |
| 管理员 | 录入/编辑/删除流水、管理分类、导入导出数据、配置账户 |
| 普通用户 | 查看流水与图表、按条件筛选统计 |

## 安全提示

> ⚠️ **重要**：本应用采用 **前端模拟登录**，仅用于区分角色与界面展示，**并非真正的身份鉴权**。

- 登录态、角色信息保存在浏览器本地，可被任意篡改或绕过。
- 仓库与 Pages 站点若为私有，访问控制由 GitHub 账号体系提供；一旦站点公开，任何人都可访问。
- **请勿在本应用中存储敏感数据**（如真实财务账户号、密码、身份证号等）。
- 数据存储于浏览器 `localStorage`，换设备/换浏览器不会同步，且清缓存即丢失。

## GitHub Pages 访问说明

本仓库默认通过 GitHub Pages 提供静态站点访问：

- 公开仓库 Pages 站点：`https://<owner>.github.io/account-book/`
- 私有仓库 Pages 需仓库所属账号为 GitHub 付费计划（Pro/Team/Enterprise）；若当前账号为免费计划，私有仓库 Pages 可能不可用，此时可手动将仓库设为公开后启用（本项目默认保持 private，不自动改公开）。

站点源配置为 `main` 分支根目录 `/`，推送 `main` 即触发发布。
